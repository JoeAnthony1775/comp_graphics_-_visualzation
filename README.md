This repository contains my final project for CS 330, including the 3D Scene ZIP folder and the Design Decisions document. Together, these items demonstrate my ability to design and develop a complete 3D environment using modern graphics programming techniques. 
When designing software, I start off by understanding the requirements and breaking the project into smaller bite size components. For this project, I had to identify the elements of the 3D scene objects, textures, lighting, camera controls, and how users would interact with it before writing code. I concentrated on clarity and structure so each part of the scene could be built, tested, and adjusted as needed
This project helped in strengthening my ability to design systems visually and spatially allowing me to make my ideas deployable, I learned how to think in terms of scale, placement, lighting balance, and camera perspective, which is different from your normal application development. I also improved my ability to document design decisions and explain why certain visual or technical choices were made.
I followed an iterative design process:
1.	Start with a simple scene layout
2.	Add objects incrementally
3.	Apply textures and materials
4.	Introduce lighting and camera movement
5.	Refine visuals based on testing and feedback
Each milestone built on the previous one, allowing me to adjust the design without restarting.
This approach can be applied to any complex software project. Breaking systems into smaller parts, testing early, and refining through iteration is useful in game development, simulations, UI design, and even backend systems.
I approach development by writing neat, modular code and testing often. In this project, I relied heavily on incremental builds, adding one feature at a time and verifying that it worked correctly before moving forward.
I gained experience working with:
•	Shader programming
•	Texture mapping and scaling
•	Lighting models (ambient, diffuse, specular)
•	Camera controls and transformations
I also learned how to debug visual issues, which required patience and experimentation rather than the usual console output debugging.
Iteration was critical. Small changes to lighting, texture scaling, or object placement often had big visual effects. I frequently adjusted values, tested results, and refined them until the scene looked balanced and intentional.
Over the milestones, my approach became more careful. Early on, I focused on getting things to work. By the end, I focused on polish, readability, and meeting the rubric requirements. I am now more comfortable working with unfamiliar libraries and debugging unclear issues.

Computer science gives me problem solving skills that transfer across many domains. This course showed me how programming can be used not just for logic and data processing, but also for creative and visual applications.
Computational graphics expanded my understanding of how math, programming, and visualization work together. These skills will help me in future courses involving simulations, visualization, and advanced software development.
The ability to design and implement a 3D environment is valuable in fields such as game development, simulation, training systems, and visualization tools. This project also strengthened my portfolio by showcasing a complete, well documented technical project rather than just isolated code samples.
